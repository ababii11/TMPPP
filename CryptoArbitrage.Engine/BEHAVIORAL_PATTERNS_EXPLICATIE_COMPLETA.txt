1️⃣  COMMAND PATTERN - "Comenzi în Cutii"
█████████████████████████████████████████████████████████████████████████████████

📌 CUM FUNCȚIONEAZĂ?
  • Client crează comenzi (ExecuteBuyOrderCommand, ExecuteSellOrderCommand)
  • Invoker (TradingBotInvoker) primește și execută comenzile
  • Fiecare comandă "știe" ce trebuie să facă
  • Invoker-ul ține historia comenzilor și poate face Undo

FLUXUL:
  1. Client crează o comandă (BUY / SELL)
  2. Client pasează comandă la Invoker
  3. Invoker execută comandă
  4. Invoker salvează în history (Stack)
  5. Poți face Undo pentru a reveni la starea anterioară

🎯 ROLUL ÎN PROIECT:
  ✓ Executa comenzi de tranzacționare (BUY/SELL)
  ✓ Undo/Redo pe operații
  ✓ Decopluează logica de execuție

📊 COMPONENTE:
  • ICommand (interfață) - contract pentru comenzi
  • ExecuteBuyOrderCommand - comandă pentru cumpărare
  • ExecuteSellOrderCommand - comandă pentru vânzare
  • TradingBotInvoker - orkestrutor care execută
  • BinanceOrderTool - receiver-ul care execută efectiv

AVANTAJE:
  ✓ Undo/Redo suport
  ✓ Planificare comenzi
  ✓ Coada de comenzi
  ✓ Logging și audit trail

DEZAVANTAJE:
  ✗ Mai multe clase
  ✗ Overhead pentru comenzi simple

2️⃣  ITERATOR PATTERN - "Parcurgere Inteligentă"
█████████████████████████████████████████████████████████████████████████████████

📌 CUM FUNCȚIONEAZĂ?
  • ExchangeCollection ține o lista de exchange-uri (Binance, Kraken, etc)
  • ExchangeIterator permite parcurgerea secvențială
  • Ascunde detaliile interne ale colecției
  • Client nu trebuie să știe cum sunt stocate exchange-urile

FLUXUL:
  1. CryptoPriceService obține iterator din ExchangeCollection
  2. Iterator itereaza prin fiecare exchange
  3. Pentru fiecare exchange: preluam prețurile
  4. Comparam prețurile între exchange-uri
  5. Detectam oportunități de arbitraj

🎯 ROLUL ÎN PROIECT:
  ✓ Parcurge toți exchange-urile
  ✓ Preluă prețuri de la fiecare
  ✓ Ascunde implementarea internă

📊 COMPONENTE:
  • IIterator (interfață) - HasNext(), Next(), Reset()
  • IExchangeCollection (interfață) - CreateIterator()
  • ExchangeCollection - lista de exchange-uri
  • ExchangeIterator - pozitie curenta + logica iterării

AVANTAJE:
  ✓ Decuplează structura de iterație
  ✓ Suporta mai multe iteratori
  ✓ Ascunde detalii interne

DEZAVANTAJE:
  ✗ Overhead mic pentru colecții mici
  ✗ Complexitate suplimentară

Nu este foarte vizual ca buton dedicat, dar este folosit în 
backend când se parcurg exchange-urile/perechile și istoricul
 (ex: lista de exchange-uri, datele care populează tabelul).
În UI îl vezi indirect prin datele agregate afișate.

3️⃣  MEMENTO PATTERN - "Salvează și Restabilește Stări"
█████████████████████████████████████████████████████████████████████████████████

📌 CUM FUNCȚIONEAZĂ?
  • ArbitrageBotStateOriginator - botul cu stare internă (balance, trades, etc)
  • SaveToMemento() - crează snapshot (fotografie) al stării curente
  • BotStateSnapshot - "memento" care ține starea
  • StateManager - ține toate snapshoturile și poate restaura

FLUXUL:
  1. Bot lucrează: executa trade-uri, schimba balance
  2. La fiecare pas important, se salvează snapshot
  3. Snapshot-ul ține: balance, strategy activă, trade-uri deschise, histórico
  4. Dacă ceva merge prost, restaurezi din snapshot
  5. Bot revine la starea salvată

🎯 ROLUL ÎN PROIECT:
  ✓ Salvează starea botului periodic
  ✓ Restaurează stare după crash/reboot
  ✓ Time-travel prin stări anterioare
  ✓ Recovery din backup

📊 COMPONENTE:
  • ArbitrageBotStateOriginator - obiect cu stare
  • BotStateSnapshot - snapshot-ul stării (imutabil)
  • StateManager - gestionează snapshoturile
  • BotTradeRecord - recorduri de trade
  • WalletTransactionEntry - historia portofelului

AVANTAJE:
  ✓ Recovery din crash
  ✓ Undo full state
  ✓ Audit trail complet
  ✓ Backup automat

DEZAVANTAJE:
  ✗ Consum memorie (fiecare snapshot e copie)
  ✗ Overhead de salvare

4️⃣  OBSERVER PATTERN - "Notificări Automate"
█████████████████████████████████████████████████████████████████████████████████

📌 CUM FUNCȚIONEAZĂ?
  • CryptoPriceService - "subject" care monitorizează prețuri
  • ArbitrageBot, LoggerService, AlertService - "observers" (ascultători)
  • Când prețuri se schimbă → CryptoPriceService notifică toți observenii
  • Fiecare observer reacționează independent

FLUXUL:
  1. CryptoPriceService se aboneaza la prețuri (din exchange-uri)
  2. ArbitrageBot, Logger, Alert se "attach-ează" (se aboneaza)
  3. Prețuri se schimbă (SimulatePriceTick)
  4. CryptoPriceService apeleaza Update() pe toți observerii
  5. Fiecare observer răspunde:
     - ArbitrageBot: "Verific oportunități de arbitraj"
     - LoggerService: "Log noile prețuri"
     - AlertService: "Trimit alert dacă spread > limita"

🎯 ROLUL ÎN PROIECT:
  ✓ Notifică în real-time când se schimbă prețuri
  ✓ Multiple acțiuni pe același eveniment
  ✓ Decuplează producător de consumatori

📊 COMPONENTE:
  • IPriceSubject - interfață subject (Attach, Detach, Notify)
  • IPriceObserver - interfață observer (Update)
  • CryptoPriceService - subject concret
  • ArbitrageBot - observer 1 (verifică arbitraj)
  • LoggerService - observer 2 (logare)
  • AlertService - observer 3 (alerte)

AVANTAJE:
  ✓ Decuplare loosely-coupled
  ✓ Reacții multiple la un eveniment
  ✓ Add/remove observeri la runtime
  ✓ Publish-Subscribe pattern

DEZAVANTAJE:
  ✗ Logica distribuita (greu de urmărit)
  ✗ Ordre notificări negarantată
  ✗ Memory leaks dacă nu faci Detach

Panoul Transaction History (dreapta sus)
Ideea este că se actualizează când apar evenimente/tranzacții noi, 
iar “ascultătorii” primesc update.

5️⃣  STRATEGY PATTERN - "Algoritmi în Cutii Interschimbabile"
█████████████████████████████████████████████████████████████████████████████████

📌 CUM FUNCȚIONEAZĂ?
  • IArbitrageStrategy - interfață pentru orice strategie
  • 3 strategii concrete: Simple, SpreadBased, Triangular
  • ArbitrageStrategyContext - managerul strategiei
  • Context.SetStrategy() - schimbă algoritm la runtime
  • Context.Analyze() - evaluează folosind strategia curentă

FLUXUL:
  1. Client alege o strategie (SpreadBasedStrategy)
  2. Context.SetStrategy(new SpreadBasedStrategy())
  3. Client apeleaza Context.Analyze(symbol, prices, fee)
  4. Context delegă la strategie.Evaluate()
  5. Strategie returnează StrategyEvaluationResult
  6. Decizii pe baza rezultatului

🎯 ROLUL ÎN PROIECT:
  ✓ Schimbă algoritm de arbitraj la runtime
  ✓ Testeaza diferite strategii
  ✓ Adapteaza la condițiile pieței

📊 COMPONENTE:
  • IArbitrageStrategy - contract strategie
  • SimpleArbitrageStrategy - arbitraj 2 exchange-uri
  • SpreadBasedStrategy - evaluare spread
  • TriangularArbitrageStrategy - arbitraj 3 criptomonede
  • ArbitrageStrategyContext - selectează/execută
  • StrategyEvaluationResult - rezultatul evaluării

AVANTAJE:
  ✓ Schimba algoritmi fără If/Else
  ✓ Testabil individual
  ✓ Extensibil (noi strategii)
  ✓ Runtime switching

DEZAVANTAJE:
  ✗ Overhead pentru cazuri simple
  ✗ Proliferare clase

================================================================================
🎓 MODUL GENERAL AL BEHAVIORAL PATTERNS
================================================================================

Behavioral patterns se concentrează pe COMPORTAMENT și COMUNICARE între obiecte.

Aspecte comune:
  1. Decuplare - separarea responsabilităților
  2. Flexibilitate - comportament dinamic
  3. Reusabilitate - componente independente
  4. Extensibilitate - ușor de adăugat noi funcționalități


================================================================================
🔗 CONEXIUNILE ÎN CRIPTO ARBITRAJ
================================================================================

FLOW-ul COMPLET al aplicației:

1. OBSERVER → Prețuri se schimbă (CryptoPriceService notifică)
   ↓
2. STRATEGY → ArbitrageBot selectează strategie de evaluare
   ↓
3. MEMENTO → Snapshot-uri periodice ale stării botului
   ↓
4. COMMAND → Generează comenzi BUY/SELL
   ↓
5. ITERATOR → Executa pe fiecare exchange
   ↓
6. LOOP → Revine la pasul 1


================================================================================
💡 SFATURI MEMORIZARE RAPIDĂ
================================================================================

🎯 Command    = Clic buton → Execută comandă → Undo posibil
🎯 Iterator   = For loop inteligent → Parcurge colecție
🎯 Memento    = Save game → Salvează stare → Load game
🎯 Observer   = Follow buton → Notificări cand se intampla ceva
🎯 Strategy   = Choose algorithm → Schimbă comportament
